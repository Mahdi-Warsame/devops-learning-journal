# Image Repositories Learned

## 1. Docker Hub

- Public Docker image registry.
- Hosts official images (e.g., Python, Ubuntu, MySQL, Nginx).
- Can store personal and organization images.
- Supports public and private repositories.

Common commands:

```bash
docker login
docker build -t username/myapp:latest .
docker push username/myapp:latest
docker pull username/myapp:latest
```

---

## 2. Amazon Elastic Container Registry (ECR)

- AWS-managed private/public container registry.
- Used to store Docker images securely inside AWS.
- Integrates with ECS, EKS, EC2, and CI/CD pipelines.

Common workflow:

```bash
aws configure
aws ecr get-login-password
docker login
docker tag
docker push
docker pull
```

# Issues Encountered

## 1. `docker-compose` command not found

### Problem

```text
bash: docker-compose: command not found
```

### Cause

Modern Docker versions use Docker Compose V2, where the command is:

```bash
docker compose
```

instead of

```bash
docker-compose
```

### Solution

Used:

```bash
docker compose up
```

instead of:

```bash
docker-compose up
```

---

## 2. AWS CLI Authentication Error

### Problem

```text
InvalidClientTokenId
```

while running AWS commands.

### Cause

The configured AWS credentials were invalid, expired, or belonged to a different AWS account.

### Troubleshooting

Checked configuration:

```bash
aws configure list
aws sts get-caller-identity
```

Verified:

- Access Key
- Secret Key
- AWS Region
- Active profile

### Solution

Updated AWS credentials using:

```bash
aws configure
```

or replaced the credentials file with valid IAM user credentials.

---

## 3. Docker Login to Amazon ECR

### Challenge

Docker cannot directly log into Amazon ECR using username/password.

### Solution

Authenticate Docker using AWS CLI:

```bash
aws ecr get-login-password \
| docker login \
--username AWS \
--password-stdin <account-id>.dkr.ecr.<region>.amazonaws.com
```

This generates a temporary authentication token for Docker.

---

## 4. Image Tagging

Initially confused about image names and tags.

Learned that before pushing to a repository, an image must be tagged correctly.

Example:

```bash
docker tag myapp:latest username/myapp:v1
```

or for ECR:

```bash
docker tag myapp:latest \
<account-id>.dkr.ecr.<region>.amazonaws.com/myapp:latest
```

---

## 5. Push Permission Errors

### Possible causes

- Not logged into Docker Hub
- Not authenticated to ECR
- Incorrect repository name
- Repository does not exist
- Missing IAM permissions

### Resolution

Verified:

- Docker login
- AWS login
- Repository name
- IAM permissions
- Correct image tag

# Lessons Learned

- Docker Hub and Amazon ECR both store Docker images but target different use cases.
- Docker Hub is commonly used for public image sharing.
- Amazon ECR integrates tightly with AWS services.
- Every image should be tagged correctly before being pushed.
- Authentication is required before pushing to any registry.
- Docker Compose simplifies building and managing multi-container applications.
- Most Docker build and push errors are caused by incorrect authentication, repository names, or image tags.
